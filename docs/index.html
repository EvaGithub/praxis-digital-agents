Architecture

Six layers — the order is code, not vibes. Full rationale in the PRD and threat model.

L0  ORCHESTRATOR   LlmAgent · ReAct loop        perceive → think → delegate
L1  PIPELINE       SequentialAgent              Scout → Audit → Report (enforced order)
L2  AUDIT TEAM     ParallelAgent                3 workers, concurrent, I/O-bound fan-out
L3  GUARDRAIL      sanitize_web_content        all scraped text delimited as UNTRUSTED
L3½ RAG            retrieve_compliance         legal claims cite retrieved docs only
L4  EVAL GATE      verify_report_claims        no narrative published unverified
L5  HUMAN GATE     outbox/ drafts only          no send tool exists — UWG Art. 3 lit. o
DEPLOY             adk web · Vertex AI Agent Engine

Case study

The practice that started it all — excellent doctors, four languages on staff, one broken website.

8/90Grade F

hautarzt-herisau.ch — HTTP-only, not mobile-responsive, no online booking, no Google Business profile, single language. The pipeline audited it, wrote a verified German narrative, generated the report and demo site above, and drafted the outreach email — in ~15 seconds, then stopped and waited for a human. That stop is the point.

Security & evaluation

Aligned with Google's Secure Vibe Coding Agent framework — details in the threat model.

Guardrails

  • Injection sanitizer on all scraped content (Pillar 3/4)
  • Zero Ambient Authority: email sending structurally impossible (Pillar 5)
  • Strict tool boundaries — no Confused Deputy paths
  • Append-only trajectory log per lead (Pillar 6/7)

Evals — 12/12 passing

  • 5× injection resistance (attacks flagged, benign German passes)
  • 3× claim verification (hallucinated findings caught)
  • 2× retrieval relevance (right law for right failure)
  • 2× trajectory order (verify-before-publish, per lead)
  • Reporter self-repair loop: rewrite on eval failure, max 2 retries
